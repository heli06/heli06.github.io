JspPage

JspPage继承了Servlet，它本质上还是Servlet；它在web.xml里的注册方式和Servlet是很类似的。

注意servlet-mapping是可以省略的，因为Jsp实现了自动部署。

如果jsp-file的路径写在了WEB-INF下，那么它不能直接访问；只能通过url-pattern的路径或是间接的转发来使用。

隐含对象

以下九个隐含对象(implicit objects)已经预先声明，不必重复声明，也不能声明同名的变量：

  • HttpServletRequest request
  • HttpServletResponse response
  • HttpSession session（当session被设置为false时不可用）
  • ServletConfig config
  • ServletContext application
  • JspWriter out
  • 类型不明的 page(类似this，指向内存中的该jsp对象)
  • PageContext pageContext（门面模式，当一个系统过于复杂，提供一个入口来使用常见函数）。这里提供了getRequest()，getSession()，getOut()等。当.jsp需要给.java传参时，传pageContext就足够了。
  • Exception exception（.jsp不需要写异常处理，所有的jsp代码都在_jspService()的try语句块中。通常没有这个对象，除非该.jsp文件里的isErrorPage被设置为true）

关于隐含对象的相关知识也可以参考菜鸟教程-JSP隐式对象
