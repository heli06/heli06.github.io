异步调用

同步调用与异步调用的区别很好理解。举个例子：老师让同学去装水，等同学把水带回来才上课，是同步调用；不管同学有没有回来都先继续上课，水来了才喝水，是异步调用。

以下是个不完整的例子，讲的是“把耗时大的任务先放入队列，然后从队列里取出任务来处理，从而实现异步调用”。

以下例子涉及LinkedBlockingQueue。这部分老师讲得零散，我蛮搜索了一篇博文，点击查看

public static final LinkedBlockingQueue<Map> queue = 
    //……………………

class A {
    public void a(int i) {
        //………………某耗时很长的过程（如大量I/O）
    }
}
class B {
    public void b() {
        //A a = new A();
        //a.a(9); 如果这样写，会在这里死等

        //可以先把消息放入消息队列
        Map map = new HashMap();
        map.put("operation", "a");
        queue.add(map);
    }
}

main() {
    new Thread() {
        public void run() {
            while(true) {
                //从队列里取消息
                Map map = (Map) queue.take();
                String op = map.get("operation");
                Integer i = (Integer) map.get("i");
                //处理消息
                if("a".equals(op)) {
                    a.a(i);
                }
            }
        }
    }
}

异步编程是有风险的，特别是有回调函数、返回的信息涉及当前状态的时候。一个解决办法是restful，无状态编程，使得返回的信息不涉及当前状态，避开安全问题。

MVC

以下内容摘自百度百科。这个词条讲得还可以。

MVC全名是Model View Controller，是模型(model)－视图(view)－控制器(controller)的缩写，一种软件设计典范，用一种业务逻辑、数据、界面显示分离的方法组织代码，将业务逻辑聚集到一个部件里面，在改进和个性化定制界面及用户交互的同时，不需要重新编写业务逻辑。MVC被独特地发展起来用于映射传统的输入、处理和输出功能在一个逻辑的图形化用户界面的结构中。

控制器是视图的观察者。典型的例子就是WINDOWS系统的图形界面。对视图的变动会被控制器观察到，控制器会更新模型，模型会通知视图状态已改变，视图会刷新。比方说删除文件，开两个窗口一起观察，从一个窗口中删掉文件后，另一个窗口中的相同文件也会消失。

老师的代码里有MVC的简单例子，路径：\7\mvc\src
有些例子是用awt实现的。

现在的趋势是V、C合一。原本有M、V、C三个类，现在只要UI类和控制器类就好了。

GUI编程

先谈谈GUI编程中几种界面布局方式：

  • 流布局：按行显示，一行摆不下，会折到下一行。组件的大小会由操作系统钦定成它认为最合适的。一行的高度由高度最大的组件决定。
  • 网格布局：可以设置成几行几类，此时整个界面被切成若干个单元格，组件放在单元格里。组件的大小和单元格的大小一样。
  • 改进型网格布局：组件不一定铺满单元格。WEB前端领域中有一个很著名的“盒模型”，感兴趣的同学可自行查阅。
  • 边界布局：将容器划分成上、下、左、右、中五块，只能放五个组件。上、下优先放，组件宽度等于容器的宽度。中间可铺满。如果组件数量超过五个，可使用面板类，组件里套组件。
  • 卡片布局：能够让多个组件共享同一个显示空间。共享空间的组件之间的关系就像一叠牌，组件叠在一起。有些组件会被其它组件压在下面。当然可改变叠放顺序。

以上布局可以嵌套使用。
