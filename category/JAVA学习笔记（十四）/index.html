JSP

先安利一下Web编程学习网站菜鸟教程，想深入学习JSP的可以参考一下。

之前的示例的写法，把页面与逻辑混在一起了，以后扩展、维护都很困难。所以要使用JSP技术。

简单来说，JSP = html + Java
html负责静态部分，Java负责动态部分。
先来个入门级的示例。把以下代码保存在名为time.jsp的文件里：

<h1>now is : <%= new java.util.Date() %></h1>

然后扔到Tomcat的examples/路径下，访问localhost:8080/examples/time.jsp，就能获取当前时间了。

.jsp文件第一次访问可能很慢，因为要转换成Java源代码。不过可以事先提前转换。转换的代码也在Tomcat下能找着（路径我没记下来，但不难找）。

JSP语法

其实我觉得看菜鸟教程的JSP语法篇比看我的笔记会更清晰些…………

脚本程序的语法格式：

<% out.println("Hello World"); %>

JSP表达式的语法格式：

<p>
   今天的日期是: <%= new java.util.Date() %>
</p>

JSP指令，用来设置与整个JSP页面相关的属性：

<%-- 定义页面的依赖属性 --%> 
<%-- 比如脚本语言、error页面、缓存需求等等 --%>
<%@ page ... %>    

<%-- 包含其他文件 --%>
<%@ include ... %>

<%-- 引入标签库的定义，可以是自定义标签 --%>
<%@ taglib ... %>

JSP的九个implicit object（隐含对象）：

  • request，HttpServletRequest类的实例
  • response，HttpServletResponse类的实例
  • out，PrintWriter类的实例，用于把结果输出至网页上。要向网页输出内容时，请优先使用它
  • session，HttpSession类的实例
  • application，ServletContext类的实例，与应用上下文有关
  • config，ServletConfig类的实例
  • pageContext，PageContext类的实例，提供对JSP页面所有对象以及命名空间的访问
  • page，类似于Java类中的this关键字
  • Exception，Exception类的对象，代表发生错误的JSP页面中对应的异常对象

JSP动作元素

JSP动作元素在请求处理阶段起作用。JSP动作元素是用XML语法写成的。
利用JSP动作可以动态地插入文件、重用JavaBean组件、把用户重定向到另外的页面、为Java插件生成HTML代码。
动作元素只有一种语法，它符合XML标准：

<jsp:action_name attribute="value" />
