多线程编程（续）

如果两个函数之间没有关系（修改不同的变量），直接用synchronized修饰会降低效率。可以这样改：

private Object lock1 = new byte[0];
private Object lock2 = new byte[0];

void a() {
    synchronized(lock1) {
        //修改变量a
    }
}
void b() {
    synchronized(lock1) {
        //修改变量a
    }
}
void c() {
    synchronized(lock2) {
        //修改变量b
    }
}

也可以这样改：

private Object lock1 = new byte[0];//就一把锁
…………
void c() {
    //不用lock2，用this
    synchronized(this) {
        //修改变量b
    }
}

注意，不要把局部变量放入锁中，会降低效率。锁主要用于保护成员变量。

阻塞与同步

在栈的例子中,原本的pop()方法并不安全。可以这样改：

//这里设栈长度为1
private char[] data = new char[1];
//………………
sychronized public void push() {
    while(index == data.length) {
        try {wait();}
        catch(Excepiton e) {}
    }

    //………………

    //notify()随机唤醒处于wait状态中的某一个
    //但一般的实现按顺序来
    notify();
}

sychronized public char pop() {
    //这里必须使用while而非if
    while(index == 0) {
        try{wait();}
        catch(Exception e) {}
    }
    index--;
    //这里得用notifyAll()
    //因为notify()可能只把同伙唤醒
    notifyAll();
    return data[index];
}

必须使用while而非if的原因：刚唤醒的线程转入运行状态前，资源可能再次被其它线程使用。
