零碎知识

  • 需要命令行的程序，在IDE中可以用run as类似的来运行。
  • 关于java- -X非标准参数：-Xms1024m，设置Java堆最小大小。-Xmx2000m，最大大小。-Xss256k，栈的大小。栈不用太大（不过递归层数多了最好设置得大一点）。因为Java的所有对象和局部变量都在堆中，栈中没多少（这点和C++不同）。
  • System这个类值得关注。System.out是static且final的。
  • 软件设计有许多原则，可以百度。比较著名的有里氏替换原则等。
  • user.home ， usr.dir：程序当前目录所在路径。

PrintStream 和 PrintWriter

如果用前文中提到的FileWriter来向文本中写数据，会很尴尬：

FileWriter fw = new FileWriter("a.txt");
fw.write(3.14);//错误，只能写字符串
fw.rite("" + 3.14 + "\r\n");//得这样，还得自己换行

用PrintWriter就舒服多了：

//上文的fw接到这里的out
PrintWriter out = new PrintWriter(fw);
out.println(3.14);

PrintStream与PrintWriter区别很小，前者是字节流，后者是字符流。

Properties

各种语言都有自己所支持的配置文件类型。比如Python，它支持 .ini文件。因为它内部有一个ConfigParser类来支持.ini 文件的读写，根据该类提供的方法程序员可以自由地操作.ini 文件。而在Java中，Java支持的是.properties文件的读写。 JDK内置的Java.util.Properties类为我们操作.properties 文件提供了便利。
Properties本质上是名值对（name-value pair），它继承了HashTable。

以下是使用范例：

Properties p = new Properties();
p.setProperty("name", "value");
p.setProperty("name", "value2");//覆盖前一个
p.setProperty("name", "value");//value重复没有关系
String v = p.getProperty("name");//value2
//文件载入方式
p.load(new FileInputStream("config.properties"));

多线程编程和匿名类

多线程编程

多线程编程的概念和意义在此不必赘言。早期的多线程通过“分时”技术制造假象，现在的多核处理器“并行”是货真价实的。

以下是线程大概的样子和不完整的使用样例：

public interface Runnable {
    public void run();
}

public class Thread implements Runnable {
    private Runnable r;
    public Thread(Runnable r) {
        this.r = r;
    }
    public void run() {
    //空函数
    }
}

//运行任务的两种办法
//第一种：创建一个实现Runnable接口的类
class A implements Runnable {
    public void run() {
    //一种实现
    }
}
class B implements Runnable {
    public void run() {
    //另一种实现
    }
}

//第二种：继承Thread，实现run方法
public class MyThread extends Thread {
    public void run() {
        //可以自己实现
    }
}

注意，在接下里的例子中，调用start()并不意味着线程马上运行，t1也不一定先于t2。线程具体的调度程序猿无权干预，这得看系统。详细知识请复习操作系统课程。

Thread t1 = new Thread(new A());
t1.start();
Thread t2 = new Thread(new B());
t2.start();
t3.start();
…………
t2.stop();//强行终止
//如果run()执行完毕，自然终止

t2.suspend();//挂起
t2.resume();//恢复

Thread.sleep(1000);//暂停1秒。
//实际上停止时间更长一点，因为停止和恢复执行是有代价的。

Thread.yield();//让步，停多久得看调度。

线程的状态划分：

  1. dead//线程死过一次后不能再运行
  2. ready
  3. running

多线程的风险

对于临界区，要将并发转为串行。比方说某个栈可能同时被两个进程访问，那么这个栈的push()和pop()要用synchronized修饰：

synchronized public void push() {}
synchronized public void pop() {}

原理：加了锁，请回忆操作系统相关内容。
