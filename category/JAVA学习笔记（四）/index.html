JAVA学习笔记（四）

  • 技术研究

零碎知识

  1. Base64的原理、编码解码方式和应用（邮箱、图片转Base64、URL等）。老师说得零散，百度吧，百度百科解释得还不错。
  2. 文件相关函数：seek()等。
  3. Java有很好的API文档，记得下载
  4. 在JAVA里，return,break等后接顺序执行语句，会报错（unreachable statement，无法到达的语句）
  5. 抛出的异常尽可能精确，提高代码可读性
  6. 覆盖函数的访问权限不能比被覆盖函数的访问权限低

private：只有自己可以访问
（default）：同包的可以访问
protected : 同包和不同包的派生类可以访问
public：都可以访问

以下是例子：

package a;
class A {
    private int j;
    protected int i;
}

package b;
class B extends a.A {
    public void a() {
        a.A a = new a.A();
        a.i = 9;//错误，不能通过父类型访问
        B b = new B();
        b.i = 9;//正确
        i = 9;
    }
}

复习内容：数据，类型，操作

  1. 算术运算，计算结果类型最低是int而非byte。

    byte b1 = 5;
byte b2 = 6;
byte b3 = 0;

b3 = b1 + b2;//错误
b3 = (byte)( b1 + b2 );//正确的写法

b1 += b2;//正确，有隐式类型转换

b1 = b2 + 5.0;//错误，此处b2进行了提升
b1 = (byte)( b2 + 5.0 );//正确的写法

  2. 浮点数不能进行移位运算。没有例子。

  3. 多态会带来风险

    class A {
    void a() {}
}
class B extends A {
    //void a() { b(); }
    void b() {}
    main() {
        A a = new B();
        a.b();//错误，警告“A类中没有b”
        ((B)a).b();//正确
        a.a();//给类b添加a()的重载后正确
    }
}

  4. 数据操作

    int i = (int) 3.3;//这样来取整不太好
//最好用Math.floor(),Math.ceil(),
//Math.round()

新内容：异常处理

错误分两类：检查（相当于C++的运行错误）和运行错误（相当于C++的逻辑错误）
一切错误的根类：Throwable
Throwable的派生类：

  1. Error(程序运行时的严重错误，可能导致虚拟机停止。此类错误不处理也无必要处理。)
  2. Exception(相对不严重的、有时是难以预测的错误，比如网络中断)

Excepion的派生类之一：RuntimeException， 以下异常均是它的派生类。
这些异常可通过程序猿提高编程技巧解决。

//编译器不管运行时异常，写了也没用
class Student {
    String id, name;
    int age;
    public void study() {

    }
    public static void main(String[] args) {
        Student s1 = new Student();
        s1.age = 18;
        //if( s1 != null )
        s1 = null;//NullPointerException, 空指针异常
        s1.age++;

        int i = 5, j = 0;
        //if( j != 0 )
        i /= j;//ArithmeticException, 算数运算异常

        Object o = new int[0];
        //if( o instanceof Student )
        s1 = (Student) o;//ClassCastException, 类型转换异常

        //if( args.length > 1 )
        System.out.println(args[1]);
        //IndexOutOfBoundsException, 下标越界异常
    }
}

但是有一类异常：checked exception，检查类异常，与程序猿无关（设备突然损坏等）。
比如IOException

//当程序里可能存在检查性异常，编译不通过
import java.io.*;

public class ReadFile {

    //第二种异常检测方法：写在函数后
    //main() throws AException，BException{  

    public static void main(String[] args) {
    //第一种异常检测方法：try
        try {
            //也可以这样抛
            //throw new Exception();

            FileInputStream fis = new FileInputStream("a.dat");
            int c = fis.read();
            fis.close();
        }
        catch(FileNotFoundException e) {}
        catch(IOException e) {}
        //如果以下catch放在最前，以上两个失效
        catch(Exception e) {
            //可以弹窗或记日志
        }
        //也可以这样写
        /*catch(Exceptione){
            if(IOException){
            …………
            }else{
            …………
            }
        }*/

        //finally会在catch执行完后执行，
        //除非强制退出System.exit()等。
        //finally和catch不一定非得有
        finally {

        }
    }
}

常见的处理异常的手段：

class Student {
    void a() throws AException {
        …………
    }
}

void teach() throws BException {
    try {
        new Student().buy();
    }
    catch(Exception e) {
        while(e != null) {
            System.out.println(e.getMessage());
            e.getCause();
        }
    }
    //也能多重抛出
    /*catch(AException e) {
        throw new BException(e);
    }*/
}

一道例题，讲的是：覆盖函数不允许抛出被覆盖函数没有声明的检查性异常

class A {
    void a() throws IOException {}
}
class B extends A {
    void a() throws Exception {}
    void x() {
        A a = new B();//错误，抛出的是Exception
        a.a();
    }
}

但如果是这样：

class A {
    void a() throws Exception {}
}
class B extends A {
    void a() throws IOException {}
    void x() {
        A a = new B();//正确
        a.a();
    }
}
