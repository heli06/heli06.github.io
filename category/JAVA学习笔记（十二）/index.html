JAVA学习笔记（十二）

  • 技术研究

零碎知识

  • Set、List、Map的实现值得关注
  • Java Application，Java Applet（小程序）值得关注
  • J2EE全都是抽象类或接口，没有实现。它只是一套标准，需要其他人去遵守、实现它。Tomcat、TomcatEE、JBox等部分或完整实现了J2EE。以下课程使用Tomcat进行教学。最好下载Zip包
  • “日志处理程序”值得关注
  • JavaEE的API要仔细阅读
  • HTTP协议需要仔细学习
  • 之前所学内容属于JavaSE，接下来的内容属于JavaEE

First J2EE Web Application

程序猿主要实现javax.servlet（继承了Servlet函数）下的各种函数。其中service()必须实现。

要这样编译：

javac -classpath apache-tomcat\……… HelloServlet

到这里还没有结束。要通过以下叙述的方法在Tomcat上部署网站。

在某个我记不清但打开目录就能想起来的文件夹下，建立first 文件夹。fisrt下建立WEB-INF、classes、lib三个子文件夹。 WEB-INF下建立web.xml文件。

把之前编译生成的class文件放入classes下。

XML是定义语言，它不能进行加减乘除计算。
可以在xsd文件内定xml文件的数据类型、是否有序等。
XML也是一个大坑，这里不再赘述。

通过Tomcat的bin目录下的startup和shutdown 文件来开启/关闭服务器。

项目不拷贝到Tomcat下也是可以的，在conf/server.xml里配置一下就行：

<Context path="/abc" docBase="d:\java\first"/>

手动建立工程非常麻烦，比较流行的IDE应该都有更加便捷的办法来建立JavaWeb项目。这里不做赘述，请同学们自行研究。
