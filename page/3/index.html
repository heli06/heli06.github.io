RequestDispatcher 中的 forward 和 include

阅读后文前建议先看看这篇：Servlet请求转发之RequestDispatcher

forward的特点：

RequestDispatcher rd = rd.forward(request, responce);
//后面不要继续处理该请求，因为已经转发给别的组件
//req， res的生命周期由其他组件决定

include的特点：

RequestDispatcher rd = rd.include(request, responce);
//后面将继续处理该请求
//req， res的生命周期由本组件决定

HttpSession

HttpSession通过Token（一串服务器生成的长随机字符串）实现。客户端获取Token后一般存在cookie中。如果客户端浏览器设置“阻止所有cookie”，需要会话支持的应用可能无法使用。
当然也有办法替代cookie：URL重写技术和隐藏域技术。详情可查看菜鸟教程-session跟踪，介绍了三大技术的使用方法。
有多少个用户同时访问，就有多少个HttpSession对象。如果某个session长时间没反应，会被服务器删除相关数据（间隔时间可以通过setMaxInactiveInterval(int interval)设置）。
Token、cookie等信息可以在浏览器的开发者工具中查看。
